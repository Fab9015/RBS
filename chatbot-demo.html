🤖 Chatbot embebible — Demo & Documentación

Esta página explica qué es el chatbot de árbol de conversación (también llamado playbook-based chatbot), cómo se diferencia de soluciones como chatbot-ui de GitHub, y cómo integrarlo en cualquier página del sitio.

1. ¿Por qué no usamos mckaywrigley/chatbot-ui?

chatbot-ui (github.com/mckaywrigley/chatbot-ui) es una aplicación completa, no un widget embebible. Requiere:

Requisito chatbot-ui (mckaywrigley) Este widget (RBS)
Servidor Node.js / Next.js ✗ Obligatorio ✓ No necesario
Base de datos (Supabase/Postgres) ✗ Obligatorio ✓ No necesario
API key de OpenAI/Anthropic ✗ Obligatorio (tiene costo) ✓ No necesario
Compatible con GitHub Pages (sitio estático) ✗ No ✓ Sí
Conversaciones definidas en JSON propio ~ Historial en Supabase ✓ Árbol en chatbot-flow.json
Respuestas de IA generativa ✓ Sí (GPT-4, Claude…) ✗ No (respuestas predefinidas)
Instalación con 1 línea de HTML ✗ No ✓ Sí
Conclusión: chatbot-ui es ideal para reemplazar ChatGPT en tu propio servidor. Para añadir un asistente FAQ a un sitio estático como éste, la opción correcta es un widget de árbol de conversación con JSON.

2. Cómo funciona el árbol de conversación (playbook)

El chatbot de RBS usa un archivo JSON (data/chatbot-flow.json) que define nodos. Cada nodo puede ser:

  • menú — muestra botones de opciones
  • respuesta ("type": "answer") — muestra texto final + enlace opcional

Estructura del JSON (extracto)

{
  // Mensaje de bienvenida mostrado al abrir el chat
  "welcome": "¡Hola! 👋 Soy el asistente de Reserva Bosque Sereno...",

  "nodes": {

    // Nodo MENÚ — muestra botones
    "start": {
      "message": "¿Qué tema te interesa?",
      "options": [
        { "label": "💰 Pagos y cuotas",  "next": "pagos" },
        { "label": "🏊 Amenidades",       "next": "amenidades" },
        { "label": "📞 Contacto",          "next": "contacto" }
      ]
    },

    // Nodo RESPUESTA TERMINAL — muestra texto y enlace
    "pagos-cuenta": {
      "type":    "answer",
      "message": "🏦 BANCO: BBVA\n🔢 Cuenta: 012 649 7268\n💳 CLABE: 012 010 0012 6497 2689",
      "link":    { "label": "Ir a preguntas frecuentes", "url": "preguntas-frecuentes.html" },
      "back":    "pagos"   // nodo al que vuelve el botón ⬅
    }
  }
}

Árbol de conversación actual

start ──┬── 💰 Pagos ──────────────────╮
        │   ├── ¿Cómo pago?             │
        │   ├── Datos bancarios         │ todos con botón
        │   ├── Enviar comprobante      │ ⬅ para volver
        │   ├── Identificar pago        │
        │   └── Recibo de pago          │
        │                               ╯
        ├── 🏊 Amenidades
        │   ├── Alberca ──┬── Horarios
        │   │            └── Tríptico
        │   ├── Asadores ─┬── Horarios
        │   │             └── Cómo reservar
        │   └── Salón ────┬── Cómo reservar
        │                   └── ¿Qué incluye?
        │
        ├── 📋 Reglamentos ─ (alberca / asadores / salón / mascotas / estacionamientos)
        ├── 🐾 Mascotas    ─ (reglamento / padrón / tríptico)
        ├── 🕐 Horarios    ─ (alberca / asadores / administración)
        ├── 🆘 Emergencias  (respuesta directa)
        └── 📞 Contacto   ─ (WhatsApp / correo / oficina / Condovive)

3. El chatbot está activo en todo el sitio

El widget se carga automáticamente desde includes/load-fragments.js, que es el include global que todas las páginas ya usan. No es necesario tocar ninguna otra página HTML.

El widget ya está activo en esta página y en todo el sitio. Haz clic en el botón 💬 en la esquina inferior derecha para probarlo.

4. Cómo editar o ampliar las conversaciones

1
Abre data/chatbot-flow.json en cualquier editor de texto.
2
Para cambiar un texto, edita el campo "message" del nodo correspondiente. Puedes usar \n para saltos de línea y **texto** para negritas.
3
Para añadir un tema nuevo, crea un nodo en "nodes" con un ID único y agrégalo como opción en el nodo padre. Ejemplo: 
// En el nodo "start", añadir la nueva opción:
{ "label": "🔧 Mantenimiento", "next": "mantenimiento" }

// Y crear el nodo nuevo más abajo:
"mantenimiento": {
  "type":    "answer",
  "message": "Para reportar una falla de mantenimiento, genera un ticket en Condovive.",
  "link":    { "label": "Ir a Condovive", "url": "condovive.html" },
  "back":    "start"
}
4
Guarda el archivo y recarga la página. No hay compilación ni servidores.

5. Archivos del proyecto

Archivo Descripción
data/chatbot-flow.json Editar aquí Árbol de conversación completo. Todos los textos y opciones.
scripts/chatbot-widget.js No tocar normalmente Motor del widget: carga JSON, renderiza mensajes y botones, maneja estilos y eventos.
Cualquier .html del sitio 1 línea Solo agrega <script src="scripts/chatbot-widget.js"></script> antes de cerrar el </body>.

6. Limitaciones de este enfoque

  • Sin IA Las respuestas son predefinidas; el bot no entiende preguntas escritas libremente. Para IA generativa se necesitaría una API key de OpenAI/Groq (tiene costo mensual).
  • Sin historial persistente Al cerrar el chat, la conversación se pierde. No hay base de datos.
  • Sin servidor Funciona completamente en el navegador; compatible con GitHub Pages.
